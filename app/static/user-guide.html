Universal Discovery Server User Guide

1. Sign In

Use local credentials through the login form, or click Microsoft Entra ID sign-in if SSO is configured.

Default first-run local admin is admin with password Admin123£. You must change this password immediately on first login before accessing inventory and admin features.

2. Inventory View

Use the top-right burger menu and open Inventory to review recent scan runs and discovered configuration items. You can filter by provider, item type, or free-text search.

3. Service Models View

Open Service Models to view a relationship overview generated from discovered inventory. Parent-child links are inferred from metadata such as resource groups and VPC context.

4. Admin Console

Admins can:

  • Create scan profiles with scan type, schedule interval, and JSON config.
  • Edit existing scan profiles directly from the profile list.
  • Run scan profiles on demand.
  • Create local and Entra-linked users.
  • Configure multiple Azure tenant accounts.
  • Create reusable secret references and assign them from dropdowns.

5. Azure Tenant Configuration

In Admin Console, open Cloud Accounts and click Configure Azure Tenant. The window prompts for tenant name, tenant ID, client ID, and a Client Secret Reference selected from a dropdown.

6. Scan Configuration JSON Examples

ICMP profile config:

{"target":"10.0.0.0/24","timeout_seconds":1,"concurrency":100,"max_hosts":1024}

SNMP profile config:

{"target":"10.0.0.0/24","community":{"$secret":{"provider":"env","key":"SNMP_COMMUNITY"}},"version":"2c","oid":"1.3.6.1.2.1.1.1.0"}

Azure profile config:

{"tenant_id":"...","client_id":"...","client_secret":{"$secret":{"provider":"azure_key_vault","vault_url":"https://my-vault.vault.azure.net/","name":"discovery-client-secret"}},"subscription_ids":["sub-id"]}

AWS profile config:

{"access_key_id":"...","secret_access_key":{"$secret":{"provider":"aws_secrets_manager","region":"us-east-1","secret_id":"discovery/aws","json_key":"secret_access_key"}},"regions":["us-east-1"]}

7. Secret Provider Reference Formats

Environment variable provider:

{"$secret":{"provider":"env","key":"MY_SECRET"}}

Azure Key Vault provider:

{"$secret":{"provider":"azure_key_vault","vault_url":"https://my-vault.vault.azure.net/","name":"secret-name","version":"optional-version"}}

AWS Secrets Manager provider:

{"$secret":{"provider":"aws_secrets_manager","region":"us-east-1","secret_id":"my/secret","json_key":"optional-field"}}

In the Admin Console, use Secret Reference Builder to generate these references and insert them directly into profile config JSON.

8. Security Notes

Keep production credentials in a secure secret store and avoid embedding long-lived secrets directly in profile JSON. Configure HTTPS and rotate app credentials regularly.
